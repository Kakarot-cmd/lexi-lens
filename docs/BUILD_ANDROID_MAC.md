# Local Android Build Setup — macOS

Mac companion to `docs/BUILD_ANDROID_LOCAL.md` (Windows). The build *model*
is identical — read `BUILD_PLAYBOOK.md` §2 (Two-Stage Build Model) first, it
is platform-agnostic and is the one thing that prevents the recurring
`.dev`-package Play rejection. This doc only covers what is **different on a
Mac**:

1. The build wrapper is `scripts/build-android.sh` (not `.cmd`).
2. The keystore path in `gradle.properties` uses **forward slashes**, not
   double-backslashes.
3. `keytool` lives in a different place.
4. Shell checks use `grep`, not `findstr`.

Standing rule is unchanged: **Android ships via local Gradle only — never
EAS.** iOS is a separate workstream (`docs/iOS_LOCAL_TESTFLIGHT_RUNBOOK.md`)
and is unaffected by anything here.

---

## Prerequisites (Mac)

| Tool | Where on macOS |
|---|---|
| Node.js + npm | https://nodejs.org |
| Java JDK 21 | bundled with Android Studio at `/Applications/Android Studio.app/Contents/jbr/Contents/Home` |
| Android SDK | `~/Library/Android/sdk` |
| EAS CLI | `npm install -g eas-cli` |

Make Gradle and `keytool` find the JDK. Add to `~/.zshrc`:

```sh
export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
export ANDROID_HOME="$HOME/Library/Android/sdk"
export PATH="$JAVA_HOME/bin:$ANDROID_HOME/platform-tools:$PATH"
```

Verify:

```sh
node --version
keytool -help >/dev/null 2>&1 && echo "keytool OK"
eas whoami
```

`scripts/build-android.sh` also auto-discovers `keytool` from `$JAVA_HOME` or
the Android Studio bundle if it is not on `PATH`, so the signing-verify step
works either way.

## 1. Export the upload keystore to the Mac (one-time)

The upload keystore is the cryptographic identity Play uses to verify every
AAB. It is the **same** keystore as on the Windows rig — export the identical
one, not a new key (a new key produces a different SHA1 and Play rejects it).

```sh
cd ~/projects/lexi-lens
eas credentials
```

Navigate: **Android → staging** (or production — same keystore) →
**Keystore: Manage everything needed to build your project →
Download credentials**. Save to:

```
~/lexi-lens-secrets/lexilens-upload.jks
```

Verify it is the right key before trusting it:

```sh
keytool -list -v \
  -keystore ~/lexi-lens-secrets/lexilens-upload.jks \
  -storepass <KEYSTORE_PASSWORD> | grep -i 'SHA1:'
```

**MUST print** `SHA1: 9C:EB:47:42:33:D7:BB:BB:6E:ED:FF:7B:10:84:55:54:CD:58:8B:D4`.
Any other fingerprint = wrong keystore; re-check both EAS profiles.

## 2. Configure local Gradle to use the keystore

`android/gradle.properties` is gitignored and regenerated by prebuild, so
this is restored per-machine **after every `prebuild --clean`**. Append:

```properties
# ─── Lexi-Lens upload keystore (EAS-exported) — macOS ──────────────────
LEXILENS_UPLOAD_STORE_FILE=/Users/nj/lexi-lens-secrets/lexilens-upload.jks
LEXILENS_UPLOAD_KEY_ALIAS=<KEY_ALIAS_FROM_EAS>
LEXILENS_UPLOAD_STORE_PASSWORD=<KEYSTORE_PASSWORD_FROM_EAS>
LEXILENS_UPLOAD_KEY_PASSWORD=<KEY_PASSWORD_FROM_EAS>
```

macOS-specific rules:

- **Forward slashes** in the path — `/Users/nj/...`. Do **not** use the
  Windows double-backslash form here; on macOS that path will not resolve and
  Gradle silently falls back to the debug keystore → Play "wrong key" reject.
- Use the **absolute** path (`/Users/nj/...`), not `~` — Java's
  `Properties` loader does not expand `~`.
- No quotes around values, no spaces around `=`.

## 3. Configure `.env.local`

Identical to the Windows doc — copy `.env.local.template` to `.env.local`,
fill **Section 2** (unsuffixed `EXPO_PUBLIC_*`) with **staging** values:

```
EXPO_PUBLIC_APP_VARIANT=staging
EXPO_PUBLIC_SUPABASE_URL=https://zhnaxafmacygbhpvtwvf.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIs...
EXPO_PUBLIC_SENTRY_DSN=https://...ingest.sentry.io/...
EXPO_PUBLIC_REVENUECAT_IOS_KEY=
EXPO_PUBLIC_REVENUECAT_ANDROID_KEY=
```

`EXPO_PUBLIC_SENTRY_DSN` must be non-empty — the build script fails fast
without it. (The dev USB build leaves it unset, which is why that build logs
`[Sentry] ... disabled`. A tester AAB on Internal should report crashes, so
the staging DSN belongs here.)

Verify Expo reads it:

```sh
npx expo config | grep -i supabase
```

## 4. Two flows — check the baked package first

The baked `applicationId` is frozen at `expo prebuild` time, not build time
(see `BUILD_PLAYBOOK.md` §2). Check it before every build:

```sh
grep -iE 'namespace|applicationId' android/app/build.gradle
```

- Both lines `com.navinj.lexilore` (no `.dev`) → **Flow 1** (routine).
- Either line `com.navinj.lexilore.dev`, or file missing → **Flow 2** (re-prebuild).

`scripts/build-android.sh` runs this same gate automatically and refuses to
build a `.dev` package, so a wrong variant fails in seconds, not after a
Play rejection.

### Flow 1 — routine build (package already correct)

```sh
# 1. Set versionCode = (Play App-bundle-explorer max + 1), by hand, in
#    android/app/build.gradle. Play's record is the only authoritative source.
grep -E 'versionCode' android/app/build.gradle

# 2. Build (explicit 'staging' keeps JS-side env immune to a stray APP_VARIANT)
./scripts/build-android.sh staging

# AAB: android/app/build/outputs/bundle/release/app-release.aab
```

### Flow 2 — recovery / variant fix / first build on this Mac

Use when the baked package is `.dev` (the current state after a `development`
USB build), `android/` is missing, or a native dep changed. **One session,
all steps:**

```sh
cd ~/projects/lexi-lens

# 1. Set the variant BEFORE prebuild — this is what defines applicationId.
export APP_VARIANT=staging
export EXPO_PUBLIC_APP_VARIANT=staging

# 2. Regenerate android/ with the staging package baked in (~60s).
npx expo prebuild --platform android --clean

# 3. GATE — verify the package is correct. Do not skip.
grep -iE 'namespace|applicationId' android/app/build.gradle
#    Expect both = com.navinj.lexilore (no .dev). If .dev/empty: stop, debug step 1.

# 4. Restore the 4 keystore props wiped by --clean (forward-slash path!).
#    See §2 above. Confirm exactly 4 lines:
grep LEXILENS_UPLOAD android/gradle.properties

# 5. Set versionCode = (Play max + 1) by hand (--clean reset it to a default).
grep -E 'versionCode' android/app/build.gradle

# 6. Build.
./scripts/build-android.sh staging

# 7. The script prints the signing SHA1 and compares it to the expected
#    upload key. Must say "MATCH" before you upload.
```

Every `--clean` is **one operation with three cleanups**: verify variant,
restore keystore props, reset versionCode. They are not separate problems.

## 5. Day-to-day

```sh
./scripts/build-android.sh staging bump   # bump versionCode +1, then build
./scripts/build-android.sh staging        # build current versionCode
./scripts/build-android.sh staging clean  # clean :app first (slower)
./scripts/build-android.sh staging skiptest  # skip Jest gate (known-good rebuild)
```

`bump` is safe **only** when the local versionCode is already ≥ Play's max.
After a `--clean` (Flow 2), set versionCode manually for the first build, then
`bump` is safe for subsequent iterations until the next `--clean`.

Then upload to **Play Console → Internal Testing → Create new release**.

---

## 6. iOS is unaffected — why

`scripts/build-android.sh` only ever runs `expo prebuild --platform android`
and `./gradlew`. It never touches `ios/`. The signing plugin
(`plugins/withReleaseSigningConfig.js`) uses `withAppBuildGradle`, whose
callback only fires during the Android prebuild pipeline — `expo prebuild
--platform ios` bypasses it entirely. Verification recipe in
`docs/BUILD_ANDROID_LOCAL.md` §6.

## Mac ↔ Windows differences (quick reference)

| Concern | macOS | Windows |
|---|---|---|
| Build wrapper | `scripts/build-android.sh` | `scripts\build-android.cmd` |
| Keystore path in `gradle.properties` | `/Users/nj/lexi-lens-secrets/lexilens-upload.jks` (forward slash, absolute) | `C:\\Users\\Mouni\\...\\lexilens-upload.jks` (double-backslash) |
| `keytool` | `$JAVA_HOME/bin/keytool` or Android Studio bundle | `C:\Program Files\Android\Android Studio\jbr\bin\keytool.exe` |
| Inspect `build.gradle` | `grep -iE 'namespace\|applicationId' ...` | `findstr /i "namespace applicationId" ...` |
| Stop daemons | `cd android && ./gradlew --stop` | `cd android && gradlew --stop` |
| Secrets dir | `~/lexi-lens-secrets/` | `C:\Users\Mouni\lexi-lens-secrets\` |

Everything else — the plugin, the AAB output path, the expected SHA1, the
versionCode-from-Play rule, the env-var shape — is identical across both rigs.
